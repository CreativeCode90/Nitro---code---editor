Nitro — Web Editor

A lightweight online web editor to open and manage local project folders.

Overview

Nitro lets users:

  • Copy a project path from their system and paste it into the editor.
  • View files and folders of the project in a tree-like explorer.
  • Select a file to read, edit, and save content.
  • Create files/folders and delete items.

Architecture

Frontend: React

  • Explorer UI to navigate project folders.
  • Recursive file list component.
  • Editor pane using @monaco-editor/react with syntax highlighting.

Backend: Flask

  • API endpoints for working directory, reading/writing files, creating/deleting files and folders.
  • CORS enabled for local development.

Terminal Feature

Nitro includes two types of terminals:

  • Basic file-handling terminal: list directories, create/remove files/folders, rename, chmod.
  • Compiling terminal: run Python, JS, Java code and view outputs.

The terminal is implemented from scratch in React and connects to a Flask backend.

API Endpoints

POST /choose      — set working directory
GET  /cd          — list files/folders
POST /selectFolder — list folder children
POST /ReadFile     — read file contents
POST /WriteFile    — write/save file
POST /makefile     — create a file
POST /makefolder   — create a folder
POST /delete       — delete file/folder

Frontend Components

  • Explore.jsx — explorer layout & API calls
  • ListDist.jsx — recursive file list
  • File.jsx — editor pane
  • ToolBar.jsx — toolbar for actions & theme switching
  • FileTabContext — holds open file state

Running Nitro Locally

  • Backend: `pip install flask flask-cors` → `python app.py`
  • Frontend: `npx create-react-app nitro` → `npm install axios @monaco-editor/react` → `npm start`
  • Open http://localhost:3000 and paste a project path.

Security Notes

  • Do not expose Nitro to public servers without authentication.
  • Running Nitro allows frontend access to your local filesystem.
  • Use path validation and permission checks to prevent unsafe operations.
